Задание 8)
Анализ URL ссылок из текстовых файлов

py-файл:
- Через параметры командной строки получить путь к анализируемой папке и другие параметры (например, исключения, вложенность, типы файлов и т.п.), в том числе путь к БД (если не задан путь, то предусмотреть путь по умолчанию, например, рядом с анализируемой папкой).
- Рекурсивно пройтись по полученному пути, найти все необходимые файлы и обработать их, а в консоль выводить этапы выполнения и/или прогресс выполнения, предусмотреть возможность отмены и/или паузы.
- По каждому файлу достать все ссылки и сохранить это в БД.
- После каждого запуска механизма поиска URL ссылок создавать новый файл БД.
- В БД для каждой ссылки хранить ее состояние, скриншот.
- Через параметры командной строки иметь возможность сформировать web-страничку, с помощью которой можно получить информацию о всех ссылках, скриншот, и активной гиперссылке.

ipynb-файл:
- Создать несколько типичных запросов к БД, на которых можно было бы увидеть основные возможности выборки данных, желательно с использованием NumPy, Pandas, Matplotlib и т.п..
Например:
- статистика ссылок по доменам
- поиск дубликатов ссылок

HTML-страничка:
- в свободной форме, но так, чтобы было удобно осуществлять навигацию по ссылкам с превью.

Уточнение:
- под URL ссылками понимаем любые ссылки, как на http, ftp, просто локальный диск , сетевой диск, путь по IP и т.п.
- БД создаем на SQLite, проектируем ее самостоятельно.
- Предусмотреть возможность работы с различными типами текстовых файлов.
- Предусмотреть обработку потенциальных ошибок.
- Для демонстрации возможностей использовать разные типы файлов, множество подкаталогов, различные параметры, возможные ошибки и т.п.
- Так как весь механизм может занимать длительное время, подумать о реализации через многопоточность.
